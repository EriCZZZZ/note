# 进程

> 进程就是一个正在执行程序的实例

## 进程的创建

1. 系统初始化
2. 正在运行的程序执行了创建进程的系统调用
3. 用户请求新建一个进程
4. 一个批处理作业的初始化

> unix中，通常使用 ```fork()``` 和 ```execve()``` 或者类似的系统调用来实现新建一个进程

## 进程的终止

1. 正常退出（自愿的）
2. 出错退出（自愿的）
3. 严重错误（非自愿）
4. 被其他进程杀死了（非自愿）

> ```exit()``` ```kill()``` 

## 进程的状态

1. 运行态（实际占用cpu）
2. 就绪态（可运行，因为其他进程正在运行，所以暂时停止）
3. 阻塞态（除非发生某些外部事件，否则不能发生）

```mermaid
graph LR;
	运行 -->|进程调度| 就绪;
	就绪 -->|进程调度| 运行;
	运行 -->|不能继续运行| 阻塞;
	阻塞 -->|外部事件的发生| 就绪;

```

## 进程的实现

### 进程表 *process table*

一个进程占用一个进程表项

### 进程表项 *进程控制块 PCB*

包含了进程状态的重要信息

- 进程管理
  - 寄存器
  - 程序计数器
  - etc
- 存储管理
  - 正文段指针
  - 数据段指针
  - 堆栈段指针
- 文件管理
  - 根目录
  - 工作目录
  - 文件描述符
  - 用户id
  - 组id
